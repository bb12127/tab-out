WHAT THIS BRANCH SHIPS
----------------------
A coherent UI/UX pass on top of the split-module architecture (main).
Folders, search, drag-and-drop, stable refresh, and a redesigned
duplicate-cleanup affordance.


FILES TOUCHED
-------------
  extension/index.html   — search bar, folder bar, removed redundant button
  extension/app.js       — folder state, drag-and-drop, stable order, refresh debounce
  extension/render.js    — folder SVGs, escHtml, drag handles, dupe affordances
  extension/style.css    — drag feedback, dupe pill styles, layout fixes


KEY CHANGES BY AREA
-------------------

1. FOLDERS
   - storage.js exports folder CRUD: getFolders, createFolder, deleteFolder,
     moveFolderOrder, setDomainFolder, setDeferredFolder, saveFolders.
   - Data lives under three keys: 'folders', 'domainFolderMap',
     'domainFolderOrder' ('__root__' is the unfiled bucket).
   - Folder bar (#folderBar) shows chips: All, <each folder>, +New folder, Manage.
   - Visible only when there are open tabs. Empty state shows a
     "drop here or click +" hint.
   - Manage panel (#folderManagePanel) has rename / delete / reorder.

2. DRAG-AND-DROP
   - Domain cards and saved items both have draggable=true plus a small
     drag handle (.card-drag-handle / .item-drag-handle).
   - Drop targets:
       * folder chips on the bar  -> move to that folder (or "All" = unfile)
       * folder bar empty placeholder -> auto-creates a folder + assigns
       * folder header in Open tabs / Saved -> move to folder
       * another card / item -> reorder within the bucket (insertion line shows)
   - Visual feedback:
       .dragging   -> 0.4 opacity on the source
       .drop-target-> amber dashed outline on folder targets
       .drop-before/.drop-after -> 3px amber bar at the insertion edge
   - Up/down arrow buttons removed (drag replaces them).

3. STABLE REFRESH (no more jumping cards)
   - _stableDomainOrder preserves card positions across re-renders.
     New domains are appended; removed domains drop out. No tab-count sort
     after first paint.
   - scheduleDashboardRefresh:
       * 600 ms debounce (collapses bursts of tab events)
       * skipped while user is dragging (.dragging selector exists)
       * onUpdated only triggers on URL/title changes, not status pings
   - FLIP animation in effects.js still smooths any movement.

4. DUPLICATE CLEANUP — REFRAME
   The previous "Close N duplicates" button looked like a peer of
   Save/Archive/Close (red X). Reframed to two complementary affordances:
     - PER-URL: each chip with duplicates shows a clickable amber pill
       at the RIGHT edge of the chip (after the close X): "x 2x".
       Click it -> that URL's extras close, one stays. Action sits right
       next to the indicator that announced the problem.
     - BULK: when duplicates span >= 2 different URLs, a small SOLID amber
       pill appears in the card header: "Clean up N". Solid background
       differentiates it from the neutral "N tabs open" status pill.
   Single-URL dupes intentionally don't show the bulk button — the per-row
   pill is enough.

5. UI POLISH
   - Search bar: capped max-width 520px (was full window).
   - Folder bar: inline-flex; controls flow next to chips (was big gap).
   - Action row in cards: flex-wrap + tighter padding so 3 buttons fit and
     never clip on the 280px column.
   - Removed redundant section-header "+ New folder" button (folder bar
     already has it).
   - Fixed CSS class-name mismatches (search bar, search icon, clear button)
     that previously caused the icon to render at full SVG size.


HOW TO TEST QUICKLY
-------------------
1. Reload the extension page (chrome://extensions -> reload Tab Out).
2. Folders:
     - Click "+ New folder" -> empty folder appears.
     - Drag a domain card onto the folder chip -> card files into folder.
     - Click the folder chip -> focused view shows only that folder's tabs.
3. Drag-and-drop:
     - Drag two cards on top of each other in the same folder/root -> reorder
       (amber line shows insertion edge).
     - Drag a saved item onto a folder header in the Saved column -> moves.
4. Duplicate cleanup:
     - Open the same URL twice -> "2x" pill appears at the right of its chip.
     - Click the amber 2x pill -> extras close.
     - Open >= 2 different URLs twice each -> header shows "Clean up N".
5. Stable refresh:
     - Open / close some tabs externally — cards should NOT reshuffle by
       tab count. Movement only happens for added/removed domains.


KNOWN FOLLOW-UPS / WATCH ITEMS
------------------------------
- storage.js still exports reorderDeferredItem (no longer called).
  Harmless dead export; remove next time you touch storage.js.
- _stableDomainOrder is in-memory only. After a full reload, the initial
  paint sorts by tab count; subsequent renders are stable. If user-visible
  flicker on first load becomes a complaint, persist this map to
  chrome.storage.local under e.g. 'rootDomainOrder'.
- The search bar filters open-tab cards and saved items. It does NOT
  currently filter inside individual chips (chip rows). If users ask for
  inline chip filtering, add a per-chip text-match in applySearch().
